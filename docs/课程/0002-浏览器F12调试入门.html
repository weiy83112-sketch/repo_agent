Lesson 0002

浏览器 F12 调试入门（Browser DevTools）

这一课解决一个很具体的问题：不要用 file:// 调试项目页面， 而要用本地 HTTP 服务打开页面，再用 F12 观察前端、请求和后端响应。

一句话： F12 是浏览器给开发者的检查台；你可以临时改页面、看报错、看请求、看接口返回。

1. 你看到的 file:// 报错是什么意思

你用的是这种地址： 

file:///C:/Users/weiyun/Desktop/agent/docs/首页.html

这种页面是直接从本地文件系统打开的。浏览器会把 file: 页面当成特殊安全来源， 很多跨文件访问、脚本加载、请求调试、Local Overrides 行为都会变得不稳定。

所以学习前后端调试时，不建议长期用 file://。更接近真实项目的方式是： 

http://127.0.0.1:5500/首页.html

2. 正确打开方式：本地 HTTP 服务

在项目根目录运行：

python -m http.server 5500 --directory docs

然后浏览器打开：

http://127.0.0.1:5500/首页.html

这时页面不再是 file://，而是正常的 http:// 页面。 F12 的 Network、Console、Sources 会更接近真实前端项目。

3. F12 修改页面：哪些会保存，哪些不会

操作位置 能做什么 是否保存到真实文件
Elements 临时修改 HTML 结构和 CSS 样式。 不会。刷新后消失。
Console 执行 JavaScript、查看报错、临时操作页面。 不会。只是当前页面会话。
Sources 查看和调试加载到浏览器里的文件。 默认不会保存。
Local Overrides 让浏览器把某些修改暂存在本地覆盖目录。 保存到浏览器覆盖目录，不等于修改项目源文件。
重要： F12 适合试错和观察。真正要长期保存的改动，还是应该改项目里的源文件。

4. F12 和前后端的关系

你以后做前后端项目时，最常用这几个面板：

Elements

看页面结构、CSS 是否生效、布局为什么错位。

Console

看前端报错，临时执行 JS，验证变量和函数。

Network

看前端有没有真的请求后端，接口状态码是多少，返回了什么。

Application

看 Local Storage、Session Storage、Cookie 等浏览器存储。

5. 一个前后端调试例子

假设页面点击按钮后应该请求后端：

GET http://127.0.0.1:8000/work-orders

你可以这样查：

  1. 打开 F12。
  2. 进入 Network 面板。
  3. 刷新页面或点击按钮。
  4. 找到 work-orders 请求。
  5. 看 Status Code：200、404、500 分别代表不同问题。
  6. 看 Response：确认后端到底返回了什么数据。

6. 状态码先记这几个

状态码 含义 排查方向
200 请求成功。 如果页面还是错，看前端渲染逻辑。
404 接口地址不存在。 检查 URL、路由、端口。
500 后端内部错误。 看后端终端日志。
CORS 跨域被浏览器拦截。 后端需要允许前端来源。

7. 这节课先记住

用 F12 学前后端时，优先用 http://127.0.0.1 打开页面。 Elements 改样式，Console 看报错，Network 看接口，Application 看浏览器存储。
