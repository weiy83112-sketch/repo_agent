今天正式从“学习 Agent 概念”进入“亲手实现 Agent 项目”。

项目方向是一个 CLI 本地仓库阅读 Agent。当前先使用普通 Python 把最基础的执行链路搭清楚，再逐步接入工具注册表、大模型、Agent loop 和 LangGraph。

今天完成的核心链路：

```text

-> 继续等待下一条命令
```

### 3. 实现 `--repo` 参数

已经理解：


`argparse` 读取参数，`Path.resolve()` 将相对路径转换成绝对路径，`is_dir()` 检查仓库目录是否真实存在。


CLI 已支持：

```text

```

经过真实验证，可以列出目标仓库中的 `README.md`、`src/`、`tests/` 等内容。

### 5. 实现安全的 `read_file`

`read_file` 接收：

```text
repo_path       仓库根目录
relative_path   仓库内部的具体文件路径
```

路径组合：

```text
repo_path / relative_path -> 目标文件的完整路径
```

工具使用 `is_relative_to(repo_root)` 检查目标文件必须位于仓库内部，防止通过 `../` 读取仓库外部文件。

CLI 已支持：

```text
/read <relative-path>
```

经过真实验证，可以读取目标仓库中的文本文件。

### 6. 实现工具错误处理

使用 `try/except ValueError` 接住 `read_file` 的预期错误。

已经验证：读取不存在的文件时，CLI 会显示错误信息；程序不会退出，后续 `/list` 和 `/exit` 仍可继续执行。

核心原则：

```text
一次工具调用失败，不应该导致整个 Agent 循环死亡。
```

## 今天真正理解的关系

### `--repo` 和 `read_file`

```text
--repo      选择允许工作的整个仓库
read_file   选择本次要读取的具体文件
```

类比：

```text
--repo               选择一个文件柜
/list                查看文件柜中有哪些文件
/read README.md      打开并阅读其中一个文件
```

### 普通 Python 工具和 Agent 工具

当前的工具首先是普通 Python 函数：

```text
list_files(repo_path)
read_file(repo_path, relative_path)
```

当前由 CLI 中的 `if` 判断选择工具。以后会把“选择工具”替换为大模型产生的结构化 `tool_call`，但底层 Python 工具函数仍然负责真正执行。

```text
当前：用户命令 -> if 判断 -> Python 函数
以后：用户问题 -> 大模型判断 -> tool_call -> Python 函数
```

理解了激活虚拟环境后的含义：终端仍位于项目目录，但 `python` 和 `pip` 会优先使用 `.venv` 中的解释器和依赖。


其中 `-c` 表示直接执行后面的 Python 代码，不需要单独创建 `.py` 文件。

### 10. 创建 Tool Registry

新增 `repo_agent/tools/registry.py`，建立工具名称与真实 Python 函数之间的映射：

```text
"list_files"  -> list_files
"read_file"   -> read_file
"search_text" -> search_text
```

理解了函数名后面不加括号表示保存函数本身，加括号才表示立即执行函数。

```python
"list_files": list_files    # 保存函数
"list_files": list_files()  # 立即调用函数
```

### 11. 创建 Tool Schema

为三个工具增加了模型可读的名称、描述和参数结构。

今天明确区分了三个概念：

```text
真实 Python 函数  负责真正执行工作
Tool Registry     供程序根据名称查找函数
Tool Schema       告诉大模型工具用途和参数规则
```

`repo_path` 在程序启动时通过 `--repo` 固定，由程序安全保管，不允许模型修改。

因此：

```text
list_files   模型参数为空 {}
read_file    模型提供 relative_path
search_text  模型提供 query
```

### 12. 创建 Tool Executor

新增统一工具执行器：

```python
def execute_tool(repo_path: Path, name: str, arguments: dict):
    tool = get_tool(name)
    return tool(repo_path=repo_path, **arguments)
```

理解了 `**arguments`：它会把字典拆成参数名和参数值。

例如：

```python
arguments = {"relative_path": "README.md"}
```

经过 `**arguments` 后，相当于：

```python
relative_path="README.md"
```

程序最终执行：

```python
read_file(repo_path=repo_path, relative_path="README.md")
```

三个工具已经通过执行器在真实 `source/mini-swe-agent` 仓库上完成验证。
