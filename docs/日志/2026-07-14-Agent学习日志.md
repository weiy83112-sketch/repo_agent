# 2026-07-14 Agent 学习日志

## 今日学习

学习 Agent State 中 Python 列表（`list`）和字典（`dict`）的嵌套写法，并理解消息记录为何采用该结构。

## 关键理解

### 1. 单条消息是字典

```python
{"role": "assistant", "content": "我来处理。"}
```

这是一条消息字典，包含两个键值对：

- `"role"`：消息的来源或角色，例如 `user`、`assistant`、`tool`。
- `"content"`：消息的具体内容。

### 2. 多条消息组成列表

```python
messages = [
    {"role": "user", "content": "帮我查看文件"},
    {"role": "assistant", "content": "我来处理。"},
]
```

列表按先后顺序保存多条消息；每一项都是一个消息字典，而不是列表本身拥有两个键值对。

### 3. State 通过字典保存消息列表

```python
state = {
    "messages": messages,
}
```

完整层级为：`state` 是字典，`state["messages"]` 是列表，列表中的每一项是含有 `role` 和 `content` 的字典。

```text
state（字典）
└── "messages"（列表）
    ├── 第 0 项：{"role": "user", "content": "..."}
    └── 第 1 项：{"role": "assistant", "content": "..."}
```

### 4. `append()` 追加一条消息

```python
state["messages"].append(
    {
        "role": role,
        "content": content,
    }
)
```

执行顺序：先用 `state["messages"]` 取出列表，再创建一条消息字典，最后用 `append()` 将该字典加入列表末尾。`append()` 会直接修改原列表，返回值是 `None`。
