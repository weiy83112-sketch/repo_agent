## 关键理解

### 1. 虚拟环境与当前目录不同

`(.venv)` 表示 PowerShell 已激活项目虚拟环境，它改变的是当前终端寻找 Python 和第三方包的路径，不会自动把当前目录切换到 `project/`。

### 2. `os` 不是内核

Python 的 `os` 是标准库模块，用来和操作系统交互。当前项目通过 `os.environ.get("DEEPSEEK_API_KEY")` 读取已经加载进 Python 进程的环境变量。

### 3. `.env` 不是自动生效的

`.env` 只是本地文本配置文件。`python-dotenv` 的 `load_dotenv()` 会读取它，并将变量放入当前 Python 进程的 `os.environ`。

### 4. OpenAI SDK 可以调用 DeepSeek

项目使用 OpenAI Python SDK 的调用格式，但 `base_url` 指向 `https://api.deepseek.com`，因此请求会发送到 DeepSeek，而不是 OpenAI。

### 5. `ModelRouter.complex(messages)` 的职责

`complex(messages)` 接收消息列表，使用内部 DeepSeek 客户端调用 `deepseek-v4-pro`，并返回完整 API response。未来 Agent 只调用能力名 `complex`，无需知道 API Key、Base URL 或具体模型名称。
