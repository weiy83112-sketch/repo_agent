# 2026-07-10 Agent 学习日志

## 今日主线

今天继续学习 Agent 工具调用链路，重点从“模型决定调用工具”推进到“工具注册表”和“结构化输出”。

核心理解：

```text
真实函数 = Python 代码，负责真正干活
工具注册表 = 给模型看的工具说明书
结构化输出 = 模型按固定格式表达意图
程序调度器 = 根据模型输出找到函数并执行
```

## 已理解的关键点

1. 大模型不会亲自执行工具。

模型负责判断：

```text
要不要调用工具
调用哪个工具
传什么参数
```

真正执行工具的是程序代码，例如 Python 函数。

2. 工具注册表是给模型看的说明书。

例如：

```json
{
  "name": "read_file",
  "description": "读取一个文本文件，并返回文件内容。",
  "parameters": {
    "path": {
      "type": "string",
      "description": "要读取的文件路径，例如 README.md"
    }
  }
}
```

它告诉模型：

```text
工具名是什么
工具适合做什么
需要哪些参数
参数是什么类型
```

3. 结构化输出让程序可以稳定读取模型结果。

不要只让模型说：

```text
这是一个调试问题。
```

而是让模型输出：

```json
{
  "task_type": "debug_error",
  "reason": "用户提到启动失败，需要排查错误"
}
```

这样程序可以直接读取 `task_type`，并决定下一步进入哪个 Agent 节点。

4. Tool Call 和 Tool Message 的关系。

```text
tool_call = 模型写的工具申请单
tool_message = 程序执行工具后返回给模型的结果单
```

流程：

```text
模型输出 tool_call
-> 程序读取 name 和 args
-> 程序根据 name 找到真实 Python 函数
-> 程序把 args 传给函数
-> 函数执行得到 result
-> 程序把 result 包装成 tool_message
-> tool_message 返回给模型继续分析
```
