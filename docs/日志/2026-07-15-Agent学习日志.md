## 关键理解

### 1. 实例方法中的 `self`

定义方法时需要写 `self`：

```python
def complex(self, messages, tools):
    ...
```

通过实例调用时不手动传入 `self`：

```python
router.complex(messages=messages, tools=tools)
```

Python 会自动把 `router` 作为 `self` 传入。

### 2. 参数校验应靠近输入边界

Tool Schema 是提供给模型的参数说明，Python 仍需在执行工具前检查字典结构、必填参数、多余参数和实际值类型。这样可以在错误参数进入文件路径处理之前给出明确原因。

### 3. 模型只提出工具请求

模型通过 `message.tool_calls` 给出工具名称和 JSON 参数。程序负责解析参数、查找真实函数、安全执行工具，并把结果写回 `state["messages"]`。

### 4. `json.loads()` 和 `json.dumps()`

```text
json.loads：JSON 字符串 -> Python 对象
json.dumps：Python 对象 -> JSON 字符串
```

模型的 `function.arguments` 是 JSON 字符串，需要先转成字典；列表等工具结果需要转回字符串才能成为工具消息内容。

### 5. Agent loop 的两个出口方向

```text
有 tool_calls
-> 保存模型请求
-> 执行全部工具
-> 保存工具结果
-> continue 再次调用模型

没有 tool_calls
-> 保存 assistant 最终回答
-> return 回答并结束 run_agent
```
