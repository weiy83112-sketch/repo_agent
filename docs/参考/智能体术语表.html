Agent 术语表（Agent Glossary）

主中文，保留英文。以后看文档、读源码、面试都按这里的词来对齐。

中文 English 一句话解释
智能体 Agent 能根据目标选择步骤、调用工具、观察结果并继续行动的 LLM 应用。
状态机 State Machine 把 Agent 的工作流程拆成状态、步骤和流转规则。
状态 State 一次任务执行过程中携带和更新的上下文。
消息列表 Messages 保存在 State 中的对话轨迹，记录用户输入、模型回复、工具调用和工具结果。
用户消息 HumanMessage 表示用户输入的一条消息。
模型消息 AIMessage 表示模型输出的一条消息，可以是最终回答，也可以携带 tool_call。
节点 Node 状态机中的一个处理步骤。
边 Edge 节点之间的连接规则。
条件边 Conditional Edge 根据 State 里的结果决定下一步去哪。
模型路由 Model Router 根据任务类型选择 simple、complex、planning 等模型能力。
模型提供者 Model Provider 封装具体模型厂商或模型服务的调用方式。
工具调用 Tool Calling 让模型选择工具名和参数，由程序执行外部函数、API、文件系统或命令。
工具注册表 Tool Registry 记录系统有哪些工具、每个工具做什么、需要什么参数、返回什么结果。
参数结构 Schema 描述工具参数的字段、类型和约束，让模型输出程序能理解的参数。
结构化输出 Structured Output 让模型按 schema 输出程序可解析的数据，例如 JSON、枚举值或工具调用参数。
工具调用请求 Tool Call 模型按结构化格式表达“我要调用哪个工具，以及传什么参数”。
工具消息 Tool Message 程序执行工具后，把结果按对应工具调用编号返回给模型的消息。
工具结果 Tool Result 工具执行后返回给 Agent 的真实观察结果。
检索增强生成 RAG 先检索外部知识，再把检索结果交给模型生成答案。
记忆 Memory 让 Agent 在一次或多次交互中保留有用上下文。
检查点 Checkpoint 保存 Agent 执行过程，方便恢复、调试或人工介入。
人工介入 Human-in-the-loop 在关键步骤让人确认、修改或批准 Agent 的行为。
评估 Evaluation 用测试样例、指标或人工标准判断 Agent 是否可靠。
开发者工具 DevTools 浏览器提供的调试工具，用来看页面结构、报错、网络请求和存储。
元素面板 Elements 查看和临时修改 HTML、CSS、页面布局。
控制台 Console 查看前端报错，临时执行 JavaScript。
网络面板 Network 查看前端请求了哪些接口、状态码和响应内容。
来源 Origin 浏览器判断安全边界的单位，通常由协议、域名和端口组成。
