# Project Context

## 项目

Repo Agent 是面向中大型 Python 仓库的只读代码问答与应届生作品集项目，使用 Python、DeepSeek Tool Calling、LangGraph 与 Code-Aware Hybrid RAG。当前已完成固定 Baseline、可断点续跑评测和本地混合检索问答链路，下一阶段是把 Hybrid RAG 接入固定评测进行对照。

## 当前断点

- 当前目标：完成 pytest 专项 Hybrid RAG 对比后，评估是否优化检索或进行人工正确性评分。
- 已完成：pytest 15 题离线检索；Hybrid Flash/Pro 各 15 题真实评测；Baseline vs Hybrid 专项报告；完整测试 99 个通过。
- 正在进行：等待用户选择下一阶段，不运行 Django 全量索引。
- 下一步：优先分析离线 Hybrid Recall@5 低于关键词的原因，或人工填写回答正确性 0/1/2。
- 阻塞项：Django 首次向量化资源开销过高，当前明确排除；新的付费调用需再次授权。
- 最后更新：2026-07-22。

## 必须遵守

- 真实代码只放在 `project/` 且使用英文路径；第三方仓库放在 `source/`，未经明确要求不修改。
- `docs/` 保存本地教学资料并可用中文路径；`cowork/` 只保存精简上下文。
- 不读取、输出、记录或提交真实密钥；凭据仅来自被 Git 忽略的本地 `.env`/环境变量。
- 真实付费调用或向外部模型发送源码前，说明范围并取得明确授权。
- 保持 Agent 只读边界；未经明确要求不增加写文件、删除文件或执行仓库命令能力。
- 教学时先给完整真实调用上下文；大框架由用户理解和操作，小型校验与重复胶水可直接完成并标注。
- 当前代码与测试是事实来源；历史记录冲突时先核对代码。
- 只维护本项目文件，不修改 Codex 全局设置、记忆、Skill、Plugin、MCP、Hook 或聊天历史。
- 接手、交出或交回任务时必须按 `cowork/HANDOFF.md` 更新唯一活动交接单，不并行创建多份交接状态。

## 上下文检索

- 普通、独立的小任务不读取 `cowork/`。
- 用户要求接手、继续外部 Agent 工作或交回任务时，先读取 `cowork/HANDOFF.md`，再按其中关键词定向检索 Decisions/Archive。
- 教学、继续旧任务、架构讨论或涉及历史决定时，先用 `rg` 检索 `cowork/DECISIONS.md`。
- 需要恢复已学知识、踩坑或历史工作时，再用主题名、文件名、函数名等关键词检索 `cowork/ARCHIVE.md`。
- 每次只读取最相关的 1–3 个小节；禁止为了“了解项目”全文加载两份文件。
- 没有命中时直接检查代码，不得因此全文读取归档。
- 信息冲突时遵循：当前用户要求 > `AGENTS.md` > 当前 `HANDOFF.md` > `DECISIONS.md` > `ARCHIVE.md`。

## 记忆更新

- 当前状态变化时替换“当前断点”，不要累加历史。
- 产生长期有效决定时合并更新 `cowork/DECISIONS.md`。
- 完成阶段或形成可复用知识、踩坑时，向 `cowork/ARCHIVE.md` 追加精简记录。
- checkpoint 后检查三份文件是否重复、矛盾或过长，并告诉用户可以新开 Codex 任务继续。
- 活动任务转交时替换 `HANDOFF.md`；交回时写明实际变更、验证、遗留问题和建议下一步，并同步替换本文件“当前断点”。
- 不记录秘密、猜测、未经验证的信息、原始聊天或可直接从代码获得的普通事实。
