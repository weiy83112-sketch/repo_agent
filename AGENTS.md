## 必须遵守

- 真实代码只放在 `project/` 且使用英文路径；第三方仓库放在 `source/`，未经明确要求不修改。
- `docs/` 保存本地教学资料并可用中文路径；`cowork/` 只保存精简上下文。
- 不读取、输出、记录或提交真实密钥；凭据仅来自被 Git 忽略的本地 `.env`/环境变量。
- 真实付费调用或向外部模型发送源码前，说明范围并取得明确授权。
- 保持 Agent 只读边界；未经明确要求不增加写文件、删除文件或执行仓库命令能力。
- 教学时先给完整真实调用上下文；大框架由用户理解和操作，小型校验与重复胶水可直接完成并标注。
- 当前代码与测试是事实来源；历史记录冲突时先核对代码。
- 只维护本项目文件，不修改 Codex 全局设置、记忆、Skill、Plugin、MCP、Hook 或聊天历史。

## 上下文检索

- 普通、独立的小任务不读取 `cowork/`。
- 教学、继续旧任务、架构讨论或涉及历史决定时，先用 `rg` 检索 `cowork/DECISIONS.md`。
- 需要恢复已学知识、踩坑或历史工作时，再用主题名、文件名、函数名等关键词检索 `cowork/ARCHIVE.md`。
- 每次只读取最相关的 1–3 个小节；禁止为了“了解项目”全文加载两份文件。
- 没有命中时直接检查代码，不得因此全文读取归档。
- 信息冲突时遵循：当前用户要求 > `AGENTS.md` > `DECISIONS.md` > `ARCHIVE.md`。

## 记忆更新

- 当前状态变化时替换“当前断点”，不要累加历史。
- 产生长期有效决定时合并更新 `cowork/DECISIONS.md`。
- 完成阶段或形成可复用知识、踩坑时，向 `cowork/ARCHIVE.md` 追加精简记录。
- checkpoint 后检查三份文件是否重复、矛盾或过长，并告诉用户可以新开 Codex 任务继续。
- 不记录秘密、猜测、未经验证的信息、原始聊天或可直接从代码获得的普通事实。
